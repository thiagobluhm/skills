Como a economia é calculada (tudo do log, nada preenchido à mão).
  • O log de nudge grava, por sessão: growth = Messages (a conversa) e baseline = piso fixo (system+tools+memória+CLAUDE.md). total = baseline + growth.
  • Economia = Messages (growth) — a conversa que você deixa de arrastar ao dar /handover+/clear. O baseline é pago dos dois jeitos (é o piso, = o "custo de retomar"), então não entra como economia. Nada de número inventado.
  • Pie "onde vai a janela": Messages vs Fixo(baseline), somando todas as sessões — direto do log. O log junta memória+CLAUDE.md+tools num só baseline, então "Fixo" é um bloco só.
  • R$ = Messages × (preço input USD/1M do modelo) × câmbio. Tabela oficial (Opus US$5 / Sonnet US$3 / Haiku US$1 / Fable US$10). O seletor de modelo é uma alavanca "e se fosse tal modelo?"; o câmbio é seu.
  • ⚠️ Contexto re-arrastado é re-lido como input — mas re-leitura cai no cache (~0,1×), então o R$ é um teto (pior caso, sem cache).
